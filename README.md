# CS-305 Software Security Portfolio

## Artifact
Artemis Financial Practices for Secure Software Report

## Client Summary
Artemis Financial is a financial services company that required improved application security. The goal was to secure communications and protect sensitive financial data by implementing encryption and secure coding practices.

## Security Enhancements
- Implemented SHA-256 hashing for checksum verification
- Generated self-signed certificate using Java Keytool (PKCS12)
- Configured HTTPS for secure communication
- Refactored code to follow secure coding standards
- Performed dependency-check and static analysis testing

## Challenges
Configuring HTTPS and certificate management was the most challenging but also the most valuable learning experience.

## Security Layers Added
- Data integrity validation with SHA-256
- Encrypted client-server communication (HTTPS)
- Vulnerability scanning and mitigation

## Testing and Validation
- Verified application runs without errors
- Confirmed HTTPS encryption works
- Re-ran vulnerability scans after refactoring

## Tools Used
- Java Keytool
- SHA-256 (SHA-2 family)
- PKCS12 keystore
- Static code analysis
- Dependency-check reports

## Skills Demonstrated
- Secure code refactoring
- Cryptographic hashing implementation
- HTTPS configuration
- Vulnerability assessment
- Industry-standard secure development practices
